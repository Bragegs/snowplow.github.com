FAQ

What does the batch pipeline do?

With Snowplow, you can track all events from your website, mobile app, server side systems, third party systems and any type of connected device, so that you have a record of what happened, when, and to whom. The Snowplow pipeline then processes these events, which means validation, enrichment, and modeling. The processed events are then loaded into Amazon Redshift on a regular basis (i.e. in batches). This then allows you to run sophisticated analytics on your event-level data.

What does the real-time pipeline do?

The real-time pipeline supports the regular loading of events into Redshift, as well as:

  • Loading your data real-time into Elasticsearch and Kibana, for real-time reporting and dashboards
  • Publishing your event data into an Amazon Kinesis stream, for real-time processing of the data in real-time applications (e.g. personalization)

The real-time pipeline is available in open source and as a private beta on our managed service. Register now to ensure you are at the front of the queue when we release the service to the public.

What is the managed service?

The managed service gives users the option to run Snowplow as a service. We deliver rich event-level data directly into your own data warehouse without the overhead associated with running a data pipeline yourself:

  • Your events are delivered directly into your own Amazon Redshift data warehouse
  • No devops required. We setup and run the pipeline as a service
  • Ongoing support to help you instrument Snowplow trackers, model your data and analyze the data
  • Total ownership and control. All your own data stays on your AWS account
  • Hassle-free. We run the Snowplow pipeline for you, so you can concentrate on analyzing the data

You can trial the managed service – batch free for one month. The real-time pipeline is available as a private beta on our managed service.

What is included with the open source version?

We love open source. Snowplow was built on top of incredible open source projects and we believe people should be able to take our code and fork or run the Snowplow pipeline themselves. The full codebase is available on GitHub. We also provide limited support to our open source users on our user group.

The managed service gives users the option to run Snowplow as a service: Have rich event-level data delivered into your own data warehouse without the overhead associated with running a data pipeline yourself.

Does the managed service include an AWS account?

No, we believe that companies should own their data and therefore recommend that our users create their own AWS account. You remain in control, whether on the managed service or the open source version. If you decide to sign up for the managed service, we will setup, monitor and upgrade the Snowplow pipeline using credentials that you control.
